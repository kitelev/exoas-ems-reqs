# req(ems): one-click WBS prototype-subtree instantiation

## Job Story

When I have designed a repeatable WBS as a homoiconic prototype subtree (e.g. the quarterly review-sheet), I want to deploy the whole graph for one parameterised instance (employee × quarter) with a single command — so that I do not hand-clone 10 nodes, re-wire the blocker chain and fix up the links every time.

## Acceptance

- Discovers the subtree via `ems__EffortPrototype_parentEffortPrototype` (transitive).
- Derives instance classes by `Prototype`-suffix strip.
- Substitutes `{{param}}` placeholders in labels/aliases with resolved parameter labels.
- Re-maps `ems__Effort_blocker` onto cloned siblings.
- Parents children to the root instance; records `exo__Asset_prototype` provenance.
- Root gets structured `exo__Asset_relates` (person + quarter) and optional `ems__Effort_parent` (Project-classed ref) else standalone.
- Domain-agnostic; both plugin and CLI runtimes register the service.

Issue kitelev/exocortex#3881 (Gap 3).
