# req(ems): a not-done effort sorts before a done effort of equal standing

## Job Story

When I review my efforts, I want the still-open ones surfaced above the
completed ones, so finished work does not bury what still needs doing.

## Statement (Gherkin)

```gherkin
Given two efforts of equal trashed-state and equal votes
  and one is done while the other is not done
When the efforts are sorted by priority
Then the not-done effort orders before the done effort
And the done effort orders after it
```
